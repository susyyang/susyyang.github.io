03_ssh远程连接命令

发表于 2018-04-25 | 分类于 remote_connection > ssh

ssh工具

  • GUN bash
  • putty
  • gitbash

ssh连接命令（gitbash演示）

ssh命令格式

ssh开头（加参数）

查看ssh软件版本（gitbash默认自带openssh）
$ ssh -V
OpenSSH_7.7p1, OpenSSL 1.0.2o 12 Mar 2018
注意：大写V
注意：openssh包括两个部分：openssh和openssl

ssh生成密钥

ssh-keygen
需要输入密钥文件名和密码（文件名可以使用默认，密码可以为空）

ssh配置多个密钥（使用config文件）

在.ssh目录下创建名称config配置文件
编辑配置config文件，格式如下：
# 配置文件参数
# Host : Host可以看作是一个你要识别的模式，对识别的模式，进行配置对应的的主机名和ssh文件
# HostName : 要登录主机的主机名（可以省略不写）
# User : 用户（可以随便写个邮箱，使用ssh 连接的时候再指定用户）
# IdentityFile : 私钥的完整路径

ssh配置多个密钥（ssh-agent代理方式）

ssh-agent相当于一个密钥管理器
第一步：获取密钥管理器（演示使用的shell是gitbash，也是一种bash，bash作为参数）
ssh-agent bash
第二步：添加需要使用私钥key（参数是完整的路径）
ssh-add ~/.ssh/key_name
**以上两步就完成了添加当前使用的key**然后可以使用连接登录操作了

ssh测试连接

ssh -T ip/域名
输入密码（如果设置了ssh密钥密码）

ssh测试github

ssh -T github.com
或者
ssh -T git@github.com

ssh测试linux云主机

测试连接腾讯云CVM
ssh -T ubuntu@111.230.229.89

ssh远程登录（口令登录）

格式：
ssh user@host
user表示登录用户
host表示主机ip或主机名
ssh默认端口是22

修改登录端口的命令格式
ssh -p port user@host
port表示要使用的端口

ssh远程登录（密钥登录）

远程主机已经添加公钥；并且本地机器存有对应的私钥
ssh user@host
# ssh1 # ssh2 # openssh # ssh命令
gitbash获取仓库（最佳实践）

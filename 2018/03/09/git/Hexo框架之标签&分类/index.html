Hexo框架之标签&分类

发表于 2018-03-09 | 分类于 git > hexo-nexT

引子

使用github pager + hexo + next主题创建的个人有几个问题

标签页面点击出现404
分类页面点击出现404

原因就是我们并没有创建标签页面和分类页面。

创建一级分类页面

创建分类页面

gitbash命令行操作
新建一个新的页面，页面就是page，名称categories。
$ hexo new page categories
打开hexo目录下source文件夹，这时里面出现了一个categories的目录；
打开categories目录里面的index文件
配置如下：
title: 分类
date: 2018-05-10 19:10:04
type: "categories"

修改主题themes的主题配置文件

打开主题配置文件，注意不是站点的配置文件。_config.yml
配置如下：（注意空格）
menu:
   categories: /categories

分类页面之二级分类

文章书写设置（二级分类不需要手动创建分类页面）

菜单栏显示

二级分类页面，则不需要在categories下面再次创建page
需要在侧边菜单栏显示二级分类页面，则：
需要配置next主题配置文件，加上对应的项，例如
Java: /categories/一级分类或二级分类

注意这里有一个坑
​
注意分类页面和标签页面的标题：分类&标签 文件的编码格式
必须是utf-8，不然就会出现乱码。

标签页面

创建标签页面

hexo new page tags
打开tags目录里面的index文件
配置如下：
title: 标签
date: 2018-05-10 19:10:04
type: "tags"

修改主题themes的主题配置文件

打开主题配置文件，注意不是站点的配置文件。_config.yml
配置如下：（注意空格）
menu:
   tags: /tags

结束

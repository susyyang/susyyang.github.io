02对称加密_验证下载包文件是否被修改过

发表于 2018-03-12 | 分类于 md5

MD5输出128bit、SHA1输出160bit、SHA256输出256bit

MD5释义：即Message-Digest Algorithm 5（信息-摘要算法5）
MD5作用：让大容量信息在用数字签名软件签署私人密钥前被”压缩”成一种保密的格式（就是把一个任意长度的字节串变换成一定长的十六进制数字串）。

sha1释义：安全哈希算法（Secure Hash Algorithm）主要适用于数字签名标准 （Digital Signature Standard DSS）里面定义的数字签名算法（Digital Signature Algorithm DSA）。
sha1特点：不可以从消息摘要中复原信息；两个不同的消息不会产生同样的消息摘要,(但会有1x10 ^ 48分之一的机率出现相同的消息摘要,一般使用时忽略)。

sha256释义：哈希值用作表示大量数据的固定大小的唯一值。数据的少量更改会在哈希值中产生不可预知的大量更改。SHA256算法的哈希值大小为 256 位。
sha256特点：作表示大量数据的固定大小。

引子

通常我们在使用互联网的过程中，需要下载很多安装包、文件；
为了确保我们下载的安装包是原版的官方的，我们需要验证；
验证的方式是根据网站提供的md5值、sha1值、sha256值、gpg数字签名等方式
与下载安装包或者文件计算出来的值进行比较，完成验证检查的目的。

dos控制台

下面以maven 3.5.3演示
下载maven包：apache-maven-3.5.3-bin.zip
查看maven包sha1：apache-maven-3.5.3-bin.zip.sha1
查看的值为：4a4844990333e2548540729ce9ab57f52a63148d
进入安装包或文件所在地址，按住shift键，同时右键选择打开命令窗口
查看文件md5，sha1，sha256

04_ssh实践一（生成ssh密钥）

发表于 2018-03-12 | 分类于 remote_connection > ssh

git终端生成ssh密钥

gitbash命令窗口
gitbash执行命令：ssh-keygen
执行命令需要输入存放密钥的文件名，默认是id_rsa
还需要输入密钥的口令（当你使用密钥连接操作时需要使用到口令），可以为空。
生成密钥。
生成密钥的程序包含在当前githash的当前目录下，或者windows默认目录users/admin/.ssh/中。（windows上）。
git密钥包含两个部分：ssh公钥，ssh私钥。

puttygen工具生成ssh密钥

下载puttygen工具

https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html

打开puttygen工具，
输入sshkey名字key comment（可选，默认）
输入sshkey密码key passphrase（为空的话，就是不设密码，与gitbash方式差不多）
点击按钮generate
等待一会，就会自动生成ssh密钥
点击保存私钥save private key（选择保存路径和文件名，保存的文件以ppk结尾，里面包含私钥和公钥的字符串）
点击保存公钥save public key（选择保存路径和文件名，文件名以ppk结尾，里面只包含公钥的字符串）

ssh密钥简介

许多Git服务器都使用SSH公钥进行认证。
为了向Git服务器提供SSH公钥，如果某系统用户尚未拥有密钥，必须事先为其生成一份。

SSH密钥用于本地git与远程git服务器（例如github，coding，gitlab）连接交互时身份认证。
SSH密钥包含两部分

公钥：id_rsa.pub（用于远程服务器上）
私钥：id_rsa（用于本地计算机）
公钥和私钥就是两组字符串，可以存放在同一个文本文件中，也可以分开存储。
gitbash是分开两个文件存储的；
puttygen是在一个文件中存储的；

git命令详解

发表于 2018-05-08

#安装
windows git 直接去官网就行，地址如下：
https://git-scm.com

##安装后，要进行配置：

git config --global user.name "Your Name"
git config --global user.email "email@example.com"

注意 git config 命令的 –global 参数，用了这个参数，表示你这台机器上所有的 Git 仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和 Email 地址。

##在项目根目录下进行单独配置

git config user.name "Your Name"
git config user.email "email@xx.com"

git config –list 查看当前配置, 在当前项目下面查看的配置是全局配置+当前项目的配置, 使用的时候会优先使用当前项目的配置

giti dsjk dfk dsfkl dfk 
dfk ldf lkdf lk 
dfkl dsfjkl dsfjk

下载&安装

下载

从github上下载安装包
或者使用gitbash从github上clone

安装

解压之后，放在hexo路径下的theme文件夹下
将解压后的文件重命名为next（必须）
打开站点的配置文件＿config.yml
将主题theme的值改为next

部署

打开hexo路径下，打开gitbash
执行命令hexo clean
执行命令hexo g
打开本地服务hexo s 或者hexo server -p 8070（更改被占用的默认端口，改为8070）
浏览器打开localhost：8070；查看新的主题效果，符合预期，则可以提交github。
执行命令hexo d；完成提交github。

使用github pager + hexo + next主题时，创建关于我的页面

创建about me页面

gitbash命令行
hexo new page "about" 
注意命令：引号可有可无。

在hexo路径下source文件夹会生成一个about文件夹。

修改about文件的，title。

2
编辑about文件夹中的index.html文件，可以修改about页面信息。

Hexo框架之标签分类

发表于 2018-03-09

引子

使用github pager + hexo + next主题创建的个人有几个问题

标签页面点击出现404
分类页面点击出现404

原因就是我们并没有创建标签页面和分类页面。

分类页面

创建分类页面

gitbash命令行操作
新建一个新的页面，页面就是page，名称categories。
$ hexo new page categories
打开hexo目录下source文件夹，这时里面出现了一个categories的目录；
打开categories目录里面的index文件
配置如下：
title: 分类
date: 2018-05-10 19:10:04
type: "categories"

修改主题themes的主题配置文件

打开主题配置文件，注意不是站点的配置文件。_config.yml
配置如下：（注意空格）
menu:
   categories: /categories

注意这里有一个坑

注意分类页面和标签页面的标题：分类&标签 文件的编码格式
必须是utf-8，不然就会出现乱码。

标签页面

创建标签页面

hexo new page tags
打开tags目录里面的index文件
配置如下：
title: 标签
date: 2018-05-10 19:10:04
type: "tags"

修改主题themes的主题配置文件

打开主题配置文件，注意不是站点的配置文件。_config.yml
配置如下：（注意空格）
menu:
   tags: /tags

结束

Hexo框架经解

发表于 2018-03-09

Hexo准备

Hexo简介

Hexo是一个快速、简洁且高效的博客框架。
Hexo使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
Hexo是高效的静态站点生成框架，她基于 Node.js。 
通过Hexo你可以轻松地使用Markdown编写文章。
除了Markdown本身的语法之外，还可以使用Hexo提供的标签插件来快速的插入特定形式的内容。

Hexo安装

安装前准备
1.安装git
2.安装nodejs
3.选择目录，右键打开gitbash，使用gitbash命令行安装。

开始安装hexo

gitbash命令行操作
$ npm install -g hexo-cli

检查hexo安装

gitbash操作
$ hexo v

Hexo建站准备（个人博客静态站点）

本地初始化Hexo（步骤）

gitbash命令行操作
$ hexo init <folder>
说明：init 后面的参数就是本地路径。
例子：hexo init mybolger；就是在当前文件夹下面新建mybolger文件夹，存放hexo初始化的文件。
如果不加参数，那就在当前目录下初始化。

初始化文件经解（非步骤）

_config.yml网站的配置文件
package.json应用程序信息
scaffolds文件夹：模版文件夹，当您新建文章时，Hexo 会根据 scaffold 来建立文件。
source文件夹：用户资源文件夹。里面包含_post文件夹。
    1.除 _posts 文件夹之外，开头命名为 _ (下划线)的文件/文件夹和隐藏的文件将会被忽略。
    2.Markdown和HTML文件会被解析并放到public 文件夹，而其他文件会被拷贝过去。
    3.Hexo通过hexo g命令解析，生成静态网页html文件，就是存放在public文件中的。
    4.public文件夹，在刚刚初始化的时候，还没有生成。
themes文件夹。主题文件夹，hexo会根据主题来生成静态网页。

Hexo建站主体（个人博客静态站点）

hexo站点配置（步骤）
打开_conifg.yml文件，进行配置。

网站参数描述
title    网站标题
subtitle    网站副标题
description    网站描述
author    您的名字
language    网站使用的语言
timezone    网站时区。Hexo 默认使用您电脑的时区。时区列表。比如说：America/New_York, Japan, 和 UTC 。

URL
参数描述默认值
url    网址
网站存放在子目录如果您的网站存放在子目录中，例如 http://yoursite.com/blog，则请将您的 url 设为 http://yoursite.com/blog 并把 root 设为 /blog/。
root    网站根目录
permalink    文章的 永久链接 格式 :year/:month/:day/:title/
permalink_defaults    永久链接中各部分的默认值

目录
参数描述默认值
source_dir    资源文件夹，这个文件夹用来存放内容。    source
public_dir    公共文件夹，这个文件夹用于存放生成的站点文件。    public
tag_dir    标签文件夹    tags
archive_dir    归档文件夹    archives
category_dir    分类文件夹 categories
code_dir    Include code 文件夹 downloads/code
i18n_dir    国际化（i18n）文件夹 :lang
skip_render    跳过指定文件的渲染，您可使用 glob 表达式来匹配路径。

文章
参数描述默认值
new_post_name    新文章的文件名称 :title.md
default_layout    预设布局    post
auto_spacing    在中文和英文之间加入空格    false
titlecase    把标题转换为 title case false
external_link    在新标签中打开链接 true
filename_case    把文件名称转换为 (1) 小写或 (2) 大写    0
render_drafts    显示草稿    false
post_asset_folder    启动 Asset 文件夹    false
relative_link    把链接改为与根目录的相对位址    false
future    显示未来的文章    true
highlight    代码块的设置

网站扩展
theme    当前主题名称。值为false时禁用主题
deploy    部署部分的设置

配置文章模版（可选）

进入scaffolds文件夹，里面包含3个文件
draft.md
page.md
post.md
根据post.md文章模版，新建一个我们自己的，符合markdown语法。
mypost.md
后期根据需要可以创建更多的模版

配置博客主题（可选，原hexo自带默认的主题theme）

参照hexo主题之NexT

创建文章

之前的步骤，我们已经完成网站的初始化hexo init
本地初始化的网站只有一个hello world的主页。
执行下面的命令之前，需要进入hexo本地路径，
使用git remote查看当前的git仓库。
下面新建文章，
gitbash命令行操作
$ hexo new [layout] <title>
命令讲解：layout文章路径，如果在_config.yml配置文件中没有配置，就是默认的source文件夹。
命令讲解：title就是文章的标题。标题名称使用引号。
例子：$ hexo new "Java_集集合—_list"

生成静态网页（步骤，重要步骤）

如上，已经完成站点和文章的创建（虽然页面很少）
下面可以生成静态网页了

gitbash命令行操作
$ hexo generate
或简写
$ hexo g
就生成静态网页了。在public目录下，就要对应文章的html文件。
public目录结构，是根据日期生成的。

发布草稿（可选，一般不用）
​
在完成文章的撰写之后，我们可以先生成草稿
然后再生成我们最后的静态页面。
gitbash命令行操作：
$ hexo publish [layout]

本地部署测试

gitbash命令行操作
启动服务器（默认启动4000端口）
hexo server 或者简写 hexo s
更改服务器端口（改为8070）
hexo server -p 8070
浏览器访问
https://localhost:8070
就可以打开本地的网站了，里面包含我们刚刚创建的文章。

Hexo建站远程部署

Hexo部署前准备

打开配置文件_config.yml
配置
deploy:
  type: git
  repo: <repository url>
  branch: [branch]
  message: [message]
配置信息详解
type就是发布方式
repo远程仓库地址
branch选择的分支
message部署的描述信息

安装部署方式的支持库

gitbash命令行
type为git时，命令：
npm install hexo-deployer-git --save

type为heroku时，命令：
npm install hexo-deployer-heroku --save

执行部署

gitbash命令行操作
hexo deploy
或者
hexo p

结束…
